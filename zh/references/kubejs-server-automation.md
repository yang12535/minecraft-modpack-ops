# KubeJS 服务端自动化

## 目录

- 判断是否复用 KubeJS
- 设计会话级延迟任务
- 发送私聊通知
- 注册玩家命令
- 重载与验证
- 故障排查

## 复用现有脚本运行时

添加新模组之前，先检查整合包中是否已有：

- KubeJS 及其所需的 JavaScript 运行时
- `kubejs/server_scripts/`
- 现有服务端脚本及其命名约定
- 确切的 Minecraft、加载器、KubeJS 和运行时版本
- 脚本日志和重载命令

对于小型策略或通知功能，优先复用健康的服务端脚本框架。新增管理模组会扩大依赖和兼容性范围。

不要删除或重写整合包作者提供的脚本。新增一个名称明确、范围有限的文件，并将可配置文本、延迟和命令名称集中放在文件开头。

## 设计登录会话

延迟登录动作必须属于一次连接会话，而不能只绑定玩家身份。

玩家登录时：

1. 将玩家 UUID 读取为字符串
2. 生成进程内唯一的会话令牌，例如时间戳加随机值
3. 使用带命名空间的键，将令牌存入玩家持久数据
4. 按要求的延迟安排一次任务

任务执行时：

1. 按 UUID 重新查找当前在线玩家
2. 玩家已离线则停止
3. 将当前存储的令牌与捕获的令牌比较
4. 两者不同时停止，因为玩家已经重连并创建了更新的会话
5. 执行动作一次
6. 删除令牌或将该会话标记为已完成

不要在长时间延迟的回调中保留原始玩家对象。任务执行时应重新解析在线玩家。

此模型自然支持：

- 多名玩家分别计时
- 延迟到期前断开连接
- 快速重连
- 新会话开始后旧计时器才触发
- 每个完成的会话只执行一次动作

它不需要逐 tick 轮询。

## 发送私聊通知

使用玩家消息 API，而不是全服广播。

将通知集中在一个辅助函数中，使自动发送和按需命令呈现完全相同的内容。使用多个消息组件组织清晰的行和格式。

通知中不要包含：

- 除非玩家必须使用，否则不要包含私有基础设施细节
- 凭证或管理端口
- 玩家特定数据
- 没有维护者和过期计划、以后可能产生误导的临时运维信息

如果通知包含公共入口，请清楚标记主用、备用和退役状态，并在中继生命周期变化时同步更新。

## 注册按需命令

注册一个字面量玩家命令，并调用同一个通知辅助函数。

明确决定：

- 是否允许所有玩家执行
- 是否支持控制台执行
- 是否需要别名
- 是否需要冷却时间

如果实现依赖玩家命令源，应明确拒绝或妥善处理控制台执行，不要解引用不存在的玩家。

命令注册可能依赖数据包或命令注册表重载阶段。仅重载服务端脚本可能只会解析文件，而不会重建命令树。

## 重载与验证

替换现有文件时先备份，然后：

1. 在有兼容本地运行时的情况下执行 JavaScript 语法检查
2. 以正常服务账户和不可执行权限安装脚本
3. 重载 KubeJS 服务端脚本
4. 在 KubeJS 服务端日志中确认该文件
5. 要求脚本计数完整，且没有新增错误
6. 当命令注册或数据包事件需要时，执行正常的服务器资源重载
7. 以玩家上下文测试命令
8. 确认 Minecraft 和所有公共隧道仍然活跃

完整资源重载期间可能出现整合包作者脚本的配方警告。比较时间戳，将其与新脚本引入的错误分开分类。

延迟路径最有力的验收测试是：

1. 在脚本生效后加入服务器
2. 保持连接直到配置的延迟到期
3. 确认只有该玩家收到一次通知
4. 重新连接并确认计时重新开始
5. 到期前断开连接，并确认旧会话之后不会收到通知

不要为了方便测试而悄悄缩短生产延迟。只有在获得授权后，才能使用明确标记的临时测试值；完成后必须恢复并重载。

## 故障排查

- 脚本可以解析但命令未知：执行所需的服务器资源重载，并检查命令注册表错误。
- 重连后旧会话仍发送消息：存储并比较每次登录的令牌，而不是只检查 UUID。
- 离线任务抛出异常：重新解析玩家，不存在时直接返回。
- 所有玩家都收到通知：将服务器广播 API 替换为目标玩家消息 API。
- 通知持续重复：安排单次回调，不要使用服务器 tick 循环。
- 编辑后的文本没有出现：确认远端文件确实变化，重载服务端脚本，并检查新的日志时间戳。
- 出现无关配方警告：与之前的日志比较；除非新脚本修改配方，否则单独报告。

## 防护栏

- 将纯服务端自动化放在服务端脚本目录，不要放入客户端脚本目录。
- 现有脚本运行时能够安全满足需求时，不要新增模组。
- 不要用玩家名称识别会话。
- 不要在授权服务器之外持久化会话令牌。
- 支持热重载时，不要重启服务器或断开在线玩家。
- 不要仅凭脚本重载成功就声称延迟路径已经完整测试。

## 完成报告

报告脚本路径、延迟行为、重连语义、命令名称和权限、重载结果、新增脚本错误数量、服务状态，以及仍需真实玩家会话完成的验收阶段。
